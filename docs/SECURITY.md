# Security & Privacy

## Principles

1. **Local-only.** There is no network code in this application. No sync, no telemetry, no
   crash uploaders, no update pings (auto-update arrives only with signed releases, v1.0).
   The absence is verifiable — the source is public and small. Text recognition on image
   captures (issue #6, `VisionTextRecognizer`) uses Apple's on-device Vision framework
   only — no image data or recognized text ever leaves the machine.
2. **Don't record secrets by default.** Pasteboard content marked
   `org.nspasteboard.ConcealedType` (password managers), `org.nspasteboard.TransientType`,
   or `org.nspasteboard.AutoGeneratedType` is skipped.
3. **Owner-only storage and scoped encryption.** `~/Library/Application Support/Permafrost/`
   is created `0700`; the SQLite database is `0600`. When the explicit concealed-content
   opt-in is enabled, concealed **text** is AES-GCM encrypted with a 256-bit key in the
   user's Keychain. Ordinary text, images, OCR metadata, and clipboard metadata remain
   plaintext; FileVault remains strongly recommended for whole-device protection.
4. **Logs never contain clipboard content.** `os.Logger` records events and counts only.

## Recording passwords — explicit opt-in

By explicit decision of the project owner, Permafrost supports recording concealed
(password) content — **off by default**, gated behind an informed-consent prompt:

- Settings → Privacy → "Record concealed content (passwords)". Flipping it on presents:

  > **Record passwords in clipboard history?**
  > Passwords you copy will be encrypted in Permafrost's local database with a key stored
  > in your macOS Keychain. They can appear in the history panel only after deliberate
  > reveal, can be pinned, and encrypted exports can be restored only with this Mac user's
  > Keychain key. Clipboard contents are still exposed whenever they are copied or pasted.
  > [Cancel] [I Understand the Risk]

- Cancel reverts the toggle. Acknowledgment is stored; the dialog isn't shown again unless
  the setting is turned off and on again.
- Once enabled, concealed items are redacted in the list, excluded from text search, and
  reveal only on deliberate action. They can be pinned and follow the same retention rules
  (pin one and it never expires). Existing concealed text from versions before encryption is
  encrypted transactionally after the Keychain key becomes available.
- Transient/auto-generated types remain skipped regardless (they're pasteboard plumbing,
  not user content).

Residual risk the user accepts: shoulder-surfing after reveal, plaintext exposure in the
system pasteboard during copy/paste, and metadata exposure (timestamps, source app, pin state,
and a deterministic unsalted content hash). If Keychain access is pending, new concealed
captures stay only in memory and are retried when the persistent key becomes available. That hash permits offline confirmation of guessed
weak secrets for someone who can read the database. Encrypted archives contain ciphertext and
are only importable with the same Keychain key; they are not portable backups.

## OCR text in images

Settings → Images → "Recognize text in images" controls local OCR for screenshots and
copied images. When enabled, Permafrost uses Apple's on-device Vision framework; no image
data or recognized text leaves the Mac. Recognized text is stored as plaintext metadata on
the image row, participates in search, appears in preview, can be copied/pasted as text,
and is included in exports with the image manifest. It follows the parent image's retention
and pinning behavior.

Disable image OCR if you want screenshots stored only as images without searchable/plaintext
recognized text. Existing OCR metadata is not automatically deleted when the setting is
turned off.

## Threat model (MVP, honest version)

| Threat | Mitigation |
|---|---|
| Other apps reading the pasteboard | Out of scope — any macOS app can read the pasteboard; that's the platform. |
| Password-manager copies leaking into history | Skipped by default via concealed/transient types; opted-in concealed text is AES-GCM encrypted and redacted by default. |
| Another user account on the Mac | POSIX permissions (0700/0600). |
| Stolen/lost machine | FileVault (recommended loudly in README); concealed text additionally uses a Keychain-backed AES-GCM key. |
| Malicious build of Permafrost | Build from source; v1.0 adds Developer ID + notarization. |
| Data exfiltration by Permafrost itself | No network code; open source; PRs adding networking require an ADR and would be rejected against CLAUDE.md principles. |

## Permissions requested

| Permission | Why | When |
|---|---|---|
| Accessibility | Synthesize ⌘V for paste-on-select | First paste attempt; optional (copy-only fallback) |
| Input Monitoring | Register the global `⌥⌘V`-style hotkey via Carbon `RegisterEventHotKey` (ADR-005); macOS separately gates this API behind Input Monitoring (`kTCCServiceListenEvent`), discovered in ADR-014 | Requested once at launch via `IOHIDRequestAccess`, before the hotkey is registered |

Permafrost never reads keystroke content — Input Monitoring here only lets the OS notify the
app when its own registered hotkey combination fires. If denied, the global hotkey doesn't
work; the panel is still reachable from the menu-bar item. No Full Disk Access, no Screen
Recording, no camera/microphone, no network entitlements.
